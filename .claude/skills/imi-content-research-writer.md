# Content Research Writer — IMI

## Pipeline: Research → Outline → Draft

### Fase 1: Research
1. Identificar 5-8 fontes primárias (dados, papers, relatórios)
2. Extrair dados-chave e citações
3. Mapear keywords SEO (volume, dificuldade, intenção)
4. Identificar gaps no conteúdo existente da IMI

### Fase 2: Outline
```
TÍTULO: [título otimizado SEO — 50-60 chars]
META DESCRIPTION: [150-160 chars com CTA]
KEYWORDS: [primária] + [3-5 secundárias]

H1: [título]
H2: [seção 1 — hook/problema]
  H3: [subseção com dado]
H2: [seção 2 — análise/solução]
  H3: [framework ou tabela]
H2: [seção 3 — implicações]
H2: [conclusão com CTA]

INTERNAL LINKS: [3-5 páginas do site IMI]
SCHEMA: [tipo — Article, FAQPage, HowTo]
```

### Fase 3: Draft
- Gancho impactante na primeira frase
- Dados verificáveis com fonte
- Parágrafos curtos (3-4 linhas max)
- Subtítulos a cada 200-300 palavras
- Tabelas e listas para escaneabilidade
- CTA natural (consultoria, não venda)

## Formatos por Canal

### Blog IMI (1200-2000 palavras)
- Schema JSON-LD (Article + BreadcrumbList)
- Internal linking (min 3 links)
- Imagem hero + alt text descritivo
- FAQ section (3-5 perguntas para featured snippet)

### LinkedIn (300-1300 caracteres)
- Hook viral na 1ª linha
- Estrutura: HOOK → CONTEXTO → ANÁLISE → INSIGHT → CTA
- Hashtags: 3-5 relevantes
- Sem links no corpo (comentário)

### Newsletter (400-600 palavras)
- Subject line provocativo e pessoal
- 1 insight principal, bem desenvolvido
- CTA: agendar consulta, responder email

## SEO Checklist
- [ ] Title tag 50-60 chars com keyword primária
- [ ] Meta description 150-160 chars com CTA
- [ ] H1 único com keyword
- [ ] H2/H3 com keywords secundárias
- [ ] Alt text em todas as imagens
- [ ] Schema markup adequado
- [ ] Internal links (min 3)
- [ ] URL slug otimizada
